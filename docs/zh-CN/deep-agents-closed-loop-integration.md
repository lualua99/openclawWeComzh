# Deep Agents 闭环逻辑集成与架构更新说明

本文档对 OpenClaw 项目中最新引入的 **Deep Agents 闭环协作与子代理任务分发系统**的整体逻辑、验证情况以及前端页面渲染包装进行了详细梳理和总结。

此架构全面对齐了 LangChain Deep Agents 的最佳实践，实现了由主代理制定计划、下发子任务、子任务自动完成状态同步、并在前端 UI 提供实时动画与进度可视化支持的完整闭环。

---

## 1. 核心架构与逻辑闭环

完整的闭环执行链路如下，已通过自动化和手动双重验证：

1. **宏观任务感知**：用户发起了一个复杂请求（例如：“研究代码并编写文档”）。
2. **计划生成**：主代理（Manager）根据 `System Prompt` 中的 `Task Coordination` 指令，首先调用 `write_todos(action="create_plan")` 建立任务大纲（Todo List）。
3. **子代理委派**：主代理针对每个子任务，通过 `sessions_spawn` 工具派发一个 Subagent，并将任务的 `taskId` 作为参数 `associatedTaskId` 传递给子代理。
4. **状态自动更新**：子代理由于是被委派状态，其提示词让它在拿到结果后自动调用 `write_todos(action="update_todo", status="done")` 来向主任务板汇报进度。
5. **实时感知注入**：网关的运行拦截器会在每次 LLM 调用前，异步读取存储在磁盘（`.openclaw/planner/`）的最新任务状态，并动态注入到当前会话的 System Prompt（`## Current Task Plan`）中，确保主代理和其他相关代理具备全局上下文意识。
6. **收尾汇总**：主代理观察到所有分配的子任务均为 `done` 时，调用 `write_todos(action="read_plan")` 或 `complete_plan` 进行封板，总结任务结果并回复给用户。

## 2. 后端新增与修改的核心组件

### 2.1 任务池（Task Planner）增强

- **`src/agents/tools/task-planner-tool.ts`**
  - **实现逻辑**：负责拦截代理对当前任务的增、删、改、查请求，并将状态序列化保存到会话对应的本地持久化文件中。
  - **新特性**：增加了 `format=digest` 返回极简的单行摘要字符串（如 `TODO[2/4]: ① 研究 ✅ ② 编码 🔄...`）以防 Token 溢出；新增了一次性完结代理操作：`complete_plan`。

### 2.2 上下文压缩机制 (Summarization Middleware)

- **`src/agents/context-summarizer.ts`**
  - **实现逻辑**：为了防止不断交互与子代理派发带来的 Token 上下文溢出，我们实现了基于字符估算的“滑窗历史修剪”（对标 `SummarizationMiddleware`）。
  - **效用验证**：优先保留系统提示、最新的连续交互以及工具返回结果本身，对过于久远的历史平滑截断，解决超出限制报错的问题。

### 2.3 `System Prompt` 增强方案

- **`src/agents/system-prompt.ts`** & **`src/auto-reply/reply/commands-system-prompt.ts`**
  - **实现逻辑**：在主代理组装 `buildAgentSystemPrompt` 时接受从外部磁盘抽取的 `taskPlanLines` 并以强制指令形式打入其脑中：`## Task Coordination ... 制定计划 -> 分发 -> 检查 -> 汇报`。

## 3. 前端界面样式包装与渲染逻辑 (Sandbox UI & Chat UI)

除了后台状态的串联，我们在对客户直接展示的 前端页面（Sandbox 和 Chat）中，深入优化了视觉展示。

### 3.1 对话框内的工具流可视化 (`ui/src/ui/chat/tool-cards.ts`)

我们为对应的交互事件进行了重绘，脱离了生硬的纯文本显示：

- **Task Planner 动态卡片 (`write_todos`)**：
  - `create_plan` 动作：左侧带有鲜艳**紫色**边框，附带 `NEW` 胸牌和描述行。
  - `add_todo` / `update_todo`：通过检测参数状态 (`todo/in_progress/done`) 分别挂载蓝色边框并提供 `ACTIVE/DONE` 标签，以及带有删除线的已完成打钩 (`✅`) 效果。
  - `Result` 卡片：包含实时的百分比进度条动画（随着任务完成，进度条推满后将从淡紫色变为绿色）。
- **子代理委派卡片 (`sessions_spawn`)**：
  - 新增带有**橙色边框**的专属卡片风格，左侧小龙虾（🦞）头像右边会带有一圈循环呼吸光效（`spawn-pulse` CSS 动画）表明子代理正在干活。
  - 任务详情带有橘色文字 `Task ID: XXX` 及 `↗ Linked` 徽标，强提示此项委派关联到全局计划。

### 3.2 沙盒侧边栏面板 (`ui/src/ui/views/sandbox.ts`)

- 当在 Sandbox 查看运行状态时，**右侧侧边栏** 新增了一个常驻的 **📋 Task Plan** 视图。
- 视图实时展现包含全局总任务描述、绿色/紫色的进度滑块 (Progress Bar)，以及下发任务序列。
- 无缝衔接原有的 **Delegation Timeline**，实现了**从宏观规划到具体节点分派**的完全一致的 3D 监控视角。
